# 3.4: Task Detail Slide-Over Panel

## Description

Implement the slide-over panel that shows full task details when a task card is clicked. Fetches data from the API, renders metadata, acceptance criteria, execution record sections, and clickable dependency links.

## Reference Files

- `prd/prd-ui-functions.md` — UF-3: Task Detail Slide-Over Panel
- `design/tech-design.md` — Interface 5 + 6 (Task Markdown Parser + navigateToTask)
- `design/api-handbook.md` — Get Task endpoint response format
- `ui/ui-design.md` — Component 3: Task Detail Slide-Over Panel

## Acceptance Criteria

- [ ] `web/static/js/detail-panel.js` created
- [ ] Panel slides in from right (40% viewport width) when task card clicked
- [ ] Fetches task detail from `/api/projects/:id/features/:slug/tasks/:taskId`
- [ ] Displays: ID, title, status (color badge), priority (P0/P1/P2 badge), scope, estimated time, dependencies list, breaking indicator, file path, record path
- [ ] Renders acceptance criteria as checklist
- [ ] Renders execution record sections: summary, files, decisions, test results (from `RecordContent` structure)
- [ ] Shows "No execution record" when `executionRecord` is null
- [ ] Dependencies are clickable links — clicking scrolls swimlane to the referenced task and highlights it
- [ ] `navigateToTask(taskId, featureSlug)` function handles cross-feature navigation (scrolls to feature row first)
- [ ] Close via: X button, clicking outside panel, Escape key
- [ ] Panel closes cleanly with slide-out animation

## User Stories

- **Story 2**: Click blocked task → panel opens with dependency chain → each dependency clickable
- **Story 4**: Panel shows execution record with structured sections → `RecordContent` fields rendered

## Implementation Notes

- Fetch data via `fetch()` API call — don't pre-load all task details
- Highlighting: add `.highlighted` CSS class to target task card, remove after 2 seconds
- Cross-feature navigation: if target task is in a collapsed feature row, expand it first

# 3.1: CSS Styles and Status Color System

## Description

Create the base stylesheet with layout, status color coding, component styles for cards, task nodes, swimlane rows, detail panel, and activity sidebar.

## Reference Files

- `prd/prd-spec.md` — Status Color Coding table (section 5.2)
- `prd/prd-ui-functions.md` — All 4 UI function states
- `ui/ui-design.md` — Design specifications

## Acceptance Criteria

- [ ] `web/static/css/styles.css` created
- [ ] Status color variables defined: pending (grey), in_progress (blue), completed (green), blocked (red), skipped (yellow)
- [ ] CSS classes: `.status-pending`, `.status-in-progress`, `.status-completed`, `.status-blocked`, `.status-skipped`
- [ ] Priority badges: `.priority-p0`, `.priority-p1`, `.priority-p2`
- [ ] Layout: project card grid (responsive), swimlane container, SVG overlay, detail panel (right 40%), activity sidebar
- [ ] Detail panel: slide-in animation, overlay backdrop
- [ ] Activity sidebar: collapsible with transition
- [ ] Edge styles: `.edge-cross-feature` (dashed line)
- [ ] Health status indicators: `.health-active` (green), `.health-complete` (blue), `.health-stale` (grey)
- [ ] Minimum viewport: 1920x1080 (no horizontal scroll on main content)

## User Stories

- **Story 1**: Project cards with visual stats → card grid layout
- **Story 2**: Blocked tasks prominently in red → `.status-blocked` color

## Implementation Notes

- Use CSS custom properties for colors to enable theming
- Keep specificity low — avoid deep nesting
- Use flexbox for card grid, CSS grid for swimlane layout

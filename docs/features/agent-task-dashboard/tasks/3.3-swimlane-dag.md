# 3.3: Swimlane DAG Renderer

## Description

Implement the core swimlane view with dagre graph layout and SVG rendering. This is the most complex UI component — renders feature rows, phase columns, task cards with status colors, and dependency arrows.

## Reference Files

- `prd/prd-ui-functions.md` — UF-2: Feature Swimlane Board
- `design/tech-design.md` — DependencyGraph, GraphNode, GraphEdge models
- `ui/ui-design.md` — Component 2: Feature Swimlane Board

## Acceptance Criteria

- [ ] `web/static/js/swimlane.js` created
- [ ] dagre library embedded in `web/static/js/dagre.min.js`
- [ ] Feature rows rendered: one row per feature, phase columns derived from PhaseInfo
- [ ] Task cards placed in correct phase column with status color
- [ ] Task card displays: ID, truncated title (30 chars), priority badge
- [ ] Dependency arrows rendered as SVG paths between task cards
- [ ] Cross-feature arrows use dashed style (`.edge-cross-feature`)
- [ ] Features sorted: blocked tasks first, then completion % ascending, then alphabetical
- [ ] Filter controls: filter by status (checkboxes), filter by priority (checkboxes)
- [ ] Collapsible rows: toggle shows only summary bar with completion %
- [ ] Click task card → triggers detail panel open (calls `window.openDetailPanel(taskId)`)
- [ ] Renders 24 features (330 tasks) in < 1 second
- [ ] No task card overlap at 1920x1080 viewport
- [ ] Phase column headers: "Phase 1", "Phase 2", ..., "Testing" for T-test tasks

## User Stories

- **Story 2**: Blocked tasks prominently red, sorted to top → status color + sort order
- **Story 3**: Cross-feature dependency arrows → dashed SVG paths between features
- **Story 5**: Activity events scroll to task → integration point

## Implementation Notes

- Use dagre's layered layout with rankDir: "LR" (left-to-right) for phase progression
- Set node width/height based on task card dimensions before layout
- Render edges after nodes; use SVG `<path>` with marker-end for arrows
- Performance: only render edges for tasks currently in viewport
- Data injected by Go template as `window.__PROJECT_DATA__` JSON

# 3.2: Landing Page — Project Card Grid

## Description

Implement the client-side JS that renders project cards on the landing page using data embedded by the Go template.

## Reference Files

- `prd/prd-ui-functions.md` — UF-1: Project Card Grid
- `design/tech-design.md` — ProjectData model
- `ui/ui-design.md` — Component 1: Project Card Grid

## Acceptance Criteria

- [ ] `web/static/js/landing.js` created
- [ ] Renders one card per project from template-injected JSON data
- [ ] Each card shows: project name, feature count, task completion progress bar, completion %, last updated (relative time), health status indicator
- [ ] Cards are clickable — navigate to `/projects/:id`
- [ ] Loading state: skeleton cards while data loads
- [ ] Error state: warning banner for projects with invalid paths
- [ ] Empty state: "No projects configured" message when config has zero entries
- [ ] Cards sorted alphabetically by project name
- [ ] No layout shift during initial render

## User Stories

- **Story 1**: Operator sees all projects with stats → card rendering with accurate data
- **Story 7**: Invalid path shows warning → error state per project card

## Implementation Notes

- Data is injected by Go template as `window.__PROJECTS_DATA__` JSON in a `<script>` tag
- Use `document.createElement` — no framework needed for this simple rendering
- Progress bar width = completionPct%

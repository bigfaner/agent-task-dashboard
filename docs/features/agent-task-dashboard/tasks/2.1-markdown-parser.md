# 2.1: Task Markdown Parser

## Description

Implement parsers for task `.md` files (extract acceptance criteria) and execution record `.md` files (extract structured sections by headings).

## Reference Files

- `design/tech-design.md` — Interface 5: Task Markdown Parser
- `design/tech-design.md` — RecordContent model
- task-cli record template: `Z:/project/ai/forge/plugins/forge/skills/record-task/templates/template.md`

## Acceptance Criteria

- [ ] `internal/scanner/parser.go` implements `ParseTaskFile(filePath string) (*TaskFileContent, error)`
- [ ] Extracts acceptance criteria from `## Acceptance Criteria` section (lines starting with `- [ ]`)
- [ ] `ParseRecordFile(filePath string) (*RecordContent, error)` parses structured sections
- [ ] `RecordContent` fields populated from `## Summary`, `## Files Created/Modified`, `## Key Decisions`, `## Test Results` headings
- [ ] Files field parsed as `[]string` (one path per line)
- [ ] Returns empty `RecordContent` with `Raw` field populated if no headings found
- [ ] Returns error for missing files (not panic)
- [ ] Unit tests with sample markdown fixtures: task file with AC, record with all sections, record with no sections, missing file
- [ ] All tests pass

## User Stories

- **Story 4**: Panel shows structured sections (summary, files, decisions, test results) → `ParseRecordFile` provides structured data

## Implementation Notes

- Use simple line-by-line parsing — no need for a full markdown AST library
- Parse headings by detecting lines starting with `## `
- Acceptance criteria lines start with `- [ ]` or `- [x]`

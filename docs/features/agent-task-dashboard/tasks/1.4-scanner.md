# 1.4: Scanner — Filesystem Reader

## Description

Implement the Scanner that reads project directories, parses `index.json` files, and builds the in-memory data model. Uses `fs.FS` interface for testability.

## Reference Files

- `design/tech-design.md` — Interface 2: Scanner
- `design/tech-design.md` — Data Models section
- task-cli source: `Z:/project/ai/forge/task-cli/pkg/task/types.go` (reference for index.json structure)
- task-cli source: `Z:/project/ai/forge/task-cli/pkg/task/index.go`

## Acceptance Criteria

- [ ] `internal/scanner/scanner.go` implements Scanner struct with `fs.FS` field
- [ ] `NewScanner(cfg *Config) *Scanner` creates scanner using `os.DirFS("/")` for production
- [ ] `ScanAll() map[string]*ProjectData` reads all configured projects
- [ ] `ScanProject(id string) (*ProjectData, error)` reads a single project
- [ ] `Invalidate()` clears the in-memory cache
- [ ] `SortFeatures(features []FeatureData)` sorts: blocked first → completion % ascending → alphabetical
- [ ] Skips projects with invalid paths (adds to `ProjectData.Warnings`)
- [ ] Skips features with malformed `index.json` (logs warning)
- [ ] Derives `PhaseInfo` from task IDs in each feature
- [ ] Computes `HealthStatus`: "active" (has in_progress or recent completion), "complete" (all done), "stale" (no updates in 7+ days)
- [ ] Expands wildcard dependencies ("1.x") to individual task IDs
- [ ] Unit tests using `testing/fstest.MapFS`: valid project, missing features dir, malformed JSON, wildcard expansion, sort order
- [ ] Integration tests using temp directory with real index.json fixtures
- [ ] All tests pass

## User Stories

- **Story 1**: Operator sees all configured projects with accurate stats → `ScanAll` provides the data
- **Story 2**: Features with blocked tasks sort to top → `SortFeatures` implements this
- **Story 7**: Invalid path shows warning → `ProjectData.Warnings` populated

## Implementation Notes

- The `index.json` format comes from forge's breakdown-tasks skill. Reference the schema at `Z:/project/ai/forge/plugins/forge/skills/breakdown-tasks/templates/index.schema.json`
- Cache keyed by project ID (lowercased name from config)
- `fs.FS` abstraction allows unit tests to inject `fstest.MapFS` without touching real filesystem

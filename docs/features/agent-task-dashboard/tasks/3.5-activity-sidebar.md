# 3.5: Activity Sidebar

## Description

Implement the collapsible activity sidebar that shows recent task status change events across all features in the current project.

## Reference Files

- `prd/prd-ui-functions.md` — UF-4: Activity Sidebar
- `design/tech-design.md` — ActivityEvent model
- `ui/ui-design.md` — Component 4: Activity Sidebar

## Acceptance Criteria

- [ ] `web/static/js/activity.js` created
- [ ] Sidebar visible by default on the right edge of swimlane page
- [ ] Shows 50 most recent events sorted by timestamp descending
- [ ] Each event shows: timestamp, task ID (clickable), task title (truncated 40 chars), feature name, event type with color (claimed=blue, completed=green, blocked=red, skipped=yellow)
- [ ] Clicking an event scrolls swimlane to that task card and highlights it briefly
- [ ] Collapsible: collapse button hides sidebar to thin bar with blocked count badge
- [ ] Expand button reopens sidebar
- [ ] Empty state: "No activity yet" message when no tasks have status changes

## User Stories

- **Story 5**: Operator sees recent activity across features → event list with timestamps
- **Story 5**: Click event scrolls to task → click handler integration with swimlane

## Implementation Notes

- Events derived from task status + index.json mtime (activity data embedded in template)
- Sort by mtime descending; if mtimes are equal, sort by feature slug alphabetically
- Badge count: count of tasks with status "blocked" across all features

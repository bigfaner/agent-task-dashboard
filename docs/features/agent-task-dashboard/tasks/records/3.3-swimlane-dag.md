### Key Decisions
- swimlane.js is a self-contained IIFE that reads window.__INITIAL_DATA__ from Go template and fetches feature/task details via REST API calls
- Feature rows sorted: blocked tasks first, then completion % ascending, then alphabetical by slug
- Phase columns derived dynamically: Phase 1, Phase 2, Phase 3+, Testing, Other (Other hidden when no tasks with unrecognized phases)
- Dependency arrows rendered as SVG paths with cubic bezier curves; within-feature arrows solid, cross-feature arrows dashed (.edge-cross-feature)
- Dense graph simplification: cross-feature arrows collapsed to dep count badges when exceeding 50 arrows threshold
- Task card displays: ID (mono), truncated title (30 chars), priority badge, status dot with color
- Filter controls operate at feature-row granularity based on task-level matching per ui-design spec
- Window.highlightTaskCard exposed globally for detail-panel.js and activity.js integration
- Arrow endpoints use right/left card edges for horizontal flow, bottom/top edges for vertical flow
- Blocked dependencies use edge-blocked class with var(--destructive) color

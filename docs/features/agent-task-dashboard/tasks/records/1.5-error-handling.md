# Task Record: 1.5 Error Handling and Input Validation

## Summary
Implemented validation middleware for URL path parameters and error handling middleware for Gin. ValidateSlug enforces ^[a-zA-Z0-9][a-zA-Z0-9-]*[a-zA-Z0-9]$ regex on :slug params. ValidateTaskID enforces ^[a-zA-Z0-9._-]+$ and blocks path traversal (..). ErrorHandler catches scanner model errors and maps them to consistent JSON error responses with appropriate HTTP status codes (400, 404, 500).

### Key Decisions
- Regex patterns compiled at package init time (var declarations) rather than per-request for performance
- Task ID validation uses regex match plus explicit path traversal check for '..' since dots are valid in task IDs (e.g., '1.1') but '..' is not
- Error handler is post-handler middleware (uses c.Next() then checks c.Errors) to catch errors from downstream handlers
- Internal error types (ErrInvalidSlug) map to external codes (invalid_slug) in the error handler middleware, keeping model layer clean
- Gin's router naturally rejects path params containing '/' so middleware only needs to handle non-slash invalid chars

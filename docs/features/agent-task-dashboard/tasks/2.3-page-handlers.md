# 2.3: Page Handlers and HTML Templates

## Description

Implement server-side HTML rendering with Go templates for the landing page and project swimlane page. Templates provide the skeleton HTML that JS components will populate.

## Reference Files

- `design/tech-design.md` — Interface 3: Page Handlers
- `prd/prd-ui-functions.md` — UF-1, UF-2 layout descriptions
- `ui/ui-design.md` — Component layouts

## Acceptance Criteria

- [ ] `internal/handler/page.go` implements `RegisterPages(r *gin.Engine, scanner *Scanner)`
- [ ] `GET /` renders `web/templates/landing.html` with project cards data
- [ ] `GET /projects/:id` renders `web/templates/swimlane.html` with feature and task data
- [ ] `web/templates/landing.html` contains: page skeleton, container for project cards, embedded CSS link, embedded JS
- [ ] `web/templates/swimlane.html` contains: page skeleton, SVG container for DAG, detail panel placeholder, activity sidebar placeholder
- [ ] Templates use `html/template` auto-escaping (XSS protection)
- [ ] 404 page rendered for non-existent project IDs
- [ ] Integration tests: landing page returns 200 with project names, project page returns 200 with feature data, invalid project returns 404
- [ ] All tests pass

## User Stories

- **Story 1**: Operator sees project cards on landing → landing template renders project data
- **Story 2**: Operator drills into project → project template renders swimlane view

## Implementation Notes

- Templates serve as HTML shells — the actual rendering of cards, DAG, and panels is done by client-side JS
- Use `template.FuncMap` to add helper functions (e.g., completion percentage formatting, relative time)
- Pass JSON data in a `<script>` tag for JS to consume (avoids extra API calls on initial load)

# 2.4: Entry Point and Static Asset Embedding

## Description

Wire everything together in `main.go`: load config, create scanner, register routes, serve static assets via `go:embed`, and start the HTTP server.

## Reference Files

- `design/tech-design.md` — Component Diagram, Dependencies table
- `design/tech-design.md` — Architecture layer diagram

## Acceptance Criteria

- [ ] `cmd/task-dashboard/main.go` implements full startup sequence
- [ ] Config loaded from `~/.task-dashboard.yaml` or `--config` CLI flag
- [ ] Port configurable via config or `--port` CLI flag (default 8080)
- [ ] Scanner created with config
- [ ] Gin engine created with page handlers, API handlers, validation middleware
- [ ] Static assets served from embedded `web/static/` via `go:embed`
- [ ] Templates loaded from embedded `web/templates/` via `go:embed`
- [ ] Server binds to `127.0.0.1` only (not `0.0.0.0`)
- [ ] Graceful shutdown on SIGINT/SIGTERM
- [ ] `go build -o task-dashboard ./cmd/task-dashboard/` produces single binary
- [ ] Binary starts and responds to `GET /` with HTML
- [ ] Binary serves embedded static JS/CSS files

## User Stories

- **Story 7**: Operator adds project to config → config reload on restart
- **Story 6**: Agent queries API → server responds

## Implementation Notes

- Use `flag` package for CLI args (`--config`, `--port`)
- Use `os.UserHomeDir()` to resolve `~` in config path
- Embed with `//go:embed web/templates web/static` in `web/embed.go`
- Gin serves static files at `/static/` prefix

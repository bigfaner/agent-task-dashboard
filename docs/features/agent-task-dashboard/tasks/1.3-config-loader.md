# 1.3: Config Loader

## Description

Implement the YAML config file loader. Reads project list and server settings from `~/.task-dashboard.yaml` or a path specified via CLI flag.

## Reference Files

- `design/tech-design.md` — Interface 1: Config
- `design/tech-design.md` — Dependencies table (gopkg.in/yaml.v3)

## Acceptance Criteria

- [ ] `internal/config/config.go` implements `Load(path string) (*Config, error)`
- [ ] `Config` struct matches tech design: `Projects []ProjectConfig`, `Server ServerConfig`
- [ ] `ProjectConfig` has `Name` and `Path` fields with YAML tags
- [ ] `ServerConfig.Port` defaults to 8080 when omitted
- [ ] Returns `ErrConfigInvalid` for missing file or malformed YAML
- [ ] Validates that at least one project is defined
- [ ] Unit tests: valid config loads, missing file returns error, invalid YAML returns error, default port applied
- [ ] All tests pass

## User Stories

- **Story 7**: As an Operator, I want to add/remove projects by editing a config file. → Config struct and Load function enable this.

## Implementation Notes

- Use `os.ExpandEnv` to support environment variables in paths
- Consider adding `$HOME` expansion for `~` in config paths
